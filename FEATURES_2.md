# Analion — 15 фич (5 этапов + 10 LLM-фич)
## Основа: FastAPI на analysis-prompts.v2.site (порт 8101, template backend)

---

## Фаза 1 — Ядро анализатора (5 фич)

### F1. Матчер проблем → методологии
`POST /api/v1/matcher`
**Вход:** описание проблемы (текст)
**Выход:** 3-5 рекомендованных методологий с рейтингом и обоснованием
**Реализация:** keyword match + семантическая близость (TF-IDF по описаниям методологий)
**Зачем:** сейчас выбор только по ID, надо по смыслу

### F2. Примеры анализа
`GET /api/v1/examples`
**Выход:** 10 готовых разборов «проблема → метод → результат»
**Источник:** встроенные шаблоны в `data/examples.json`
**Зачем:** показать как выглядит результат без ввода своей проблемы

### F3. Граф связей методологий
`GET /api/v1/graph`
**Выход:** JSON-граф (nodes + edges) какие методы дополняют друг друга
**Реализация:** статическая карта связей (TRIZ+FMEA, SWOT+PESTLE, RICE+ICE и т.д.)
**Зачем:** видеть комбинации, а не одиночные методы

### F4. Сравнение методологий
`POST /api/v1/compare`
**Вход:** список ID (2-4 методологии)
**Выход:** таблица по критериям (сложность, время, данные, область, результат)
**Реализация:** заранее заполненные метрики для каждой методологии
**Зачем:** выбрать лучший метод под задачу

### F5. PDF-отчёт по анализу
`POST /api/v1/report`
**Вход:** проблема + методология
**Выход:** PDF-файл с анализом, шагами, рекомендациями
**Реализация:** мы уже генерим анализ через template; report = тот же анализ + reportlab/weasyprint PDF
**Зачем:** скачать результат как документ

## Фаза 2 — LLM-интеграции (10 фич)

### F6. Mistral реальный анализ (замена template)
`POST /api/v1/analyze` (с backend="mistral")
**Суть:** подключить Mistral API как бэкенд — настоящий AI-анализ вместо шаблонного
**Реализация:** MISTRAL_API_KEY в окружении, уже есть код в `runner/backends/mistral.py`
**Статус:** код есть, ключа нет

### F7. Выбор бэкенда (Mistral / Template / SambaNova)
**Суть:** пользователь выбирает — быстрый шаблонный анализ или глубокий AI
**API:** `?backend=template|mistral|sambanova` в запросе
**Реализация:** уже заложено в adapter.py, через ANALION_BACKEND env

### F8. История анализов
`GET /api/v1/history` + `POST /api/v1/analyze` сохраняет в БД
**Суть:** каждый анализ сохраняется — можно вернуться, посмотреть, сравнить
**БД:** SQLite (analion.db уже есть), таблица `analysis_history`

### F9. Глубокий анализ (multi-framework)
`POST /api/v1/deep-analyze`
**Вход:** проблема
**Выход:** анализ 3 разными методологиями + синтез (сводка)
**Реализация:** запускает matcher → анализирует каждым методом → собирает общий вывод

### F10. Резюме анализа (LLM)
`POST /api/v1/summarize`
**Вход:** сырой результат анализа
**Выход:** 3-5 предложений «суть»
**Реализация:** через LLM (Mistral) или через шаблон (template)

### F11. Чек-лист по методологии
`GET /api/v1/frameworks/{id}/checklist`
**Выход:** пошаговый чек-лист с галочками (JSON)
**Реализация:** парсинг TXT → извлечение шагов
**Зачем:** можно интегрировать с фронтендом

### F12. Оценка сложности проблемы
`POST /api/v1/problem-score`
**Вход:** описание проблемы
**Выход:** score 1-10 (сложность, срочность, неопределённость)
**Реализация:** LLM-анализ или keyword heuristics

### F13. Генерация гипотез (Brainstorm mode)
`POST /api/v1/brainstorm`
**Вход:** проблема + методология
**Выход:** 5-10 гипотез/идей по решению
**Реализация:** LLM prompt с инструкцией «сгенерируй гипотезы»

### F14. Анализ текста (загрузка файла)
`POST /api/v1/analyze-file`
**Вход:** текстовый файл (проблема, описание кейса)
**Выход:** анализ выбранной методологией
**Реализация:** читает текст из файла → передаёт в analyze

### F15. Telegram-бот поверх API (опционально)
**Суть:** тот же API, но через Telegram. Без нового поддомена.
**Реализация:** long-polling бот на этом же сервере, использует `localhost:8101/api/v1/*`
**Статус:** опционально, когда надоест curl

## Технические заметки

- Все эндпоинты = FastAPI на порту 8101 (full proxy на analysis-prompts.v2.site)
- template backend — всегда работает, без ключей
- LLM backend (Mistral) — требует MISTRAL_API_KEY в .env или env
- БД — SQLite, `analion.db` в корне сайта
- Все новые данные (примеры, граф, метрики) — JSON-файлы в `data/`
