# Five Whys + Ishikawa (Fishbone)

**Авторы:** Five Whys — Сакити Тоёда (Toyota, 1930-е); Ishikawa — Каору Исикава (1960-е)
**Тип:** Причинно-следственный анализ
**Что анализирует:** Корневые причины проблем

## Часть A: Five Whys (5 почему)

#
## 📖 ИСТОРИЯ СОЗДАНИЯ

1950-е. Япония. Сакити Тоёда, основатель Toyota, формулирует принцип: «Спрашивай "почему?" пять раз». Его сын Кийтиро внедряет это в Toyota Production System.

В 1960-х Каору Исикава дополняет метод диаграммой «рыбья кость» (fishbone diagram) — визуальным инструментом для поиска корневых причин.

Метод становится сердцем Toyota Production System и распространяется по всему миру как основа root cause analysis.

## 👤 КЛЮЧЕВЫЕ ФИГУРЫ

**Сакити Тоёда** (1867–1930) — основатель Toyota, изобрёл автоматический ткацкий станок.
**Каору Исикава** (1915–1989) — японский профессор, создатель диаграммы Исикавы и кружков качества.

## 🌍 КОНТЕКСТ ПРИМЕНЕНИЯ

- Производство: поиск корневой причины дефектов
- IT: анализ инцидентов и багов
- Управление: почему проект опоздал?

## 🏭 ИЗВЕСТНЫЕ КЕЙСЫ

**Toyota:** Пять «почему?» о покраске кузова привели не к замене робота, а к чистке форсунок раз в смену. Экономия: $500K/год на одном заводе.

---

## Алгоритм
1. Сформулируй проблему одним предложением (факт, не мнение).
   «Клиенты не проходят онбординг» → «35% новых пользователей не открывают приложение на 2-й день».

2. Спроси «Почему это произошло?» Запиши ответ.

3. К этому ответу снова спроси «Почему?» Запиши следующий уровень.

4. Повтори 5 раз (или пока не дойдёшь до корневой причины, которую можно исправить).

5. Проверь: если устранить эту причину, проблема исчезнет?

### Правила
- Не перескакивай: каждый ответ — из предыдущего.
- Не обвиняй людей: ищи причины в процессе/системе.
- Может быть несколько веток: одна проблема → несколько «почему».

### Пример
Проблема: сайт упал.
1. Почему? → Сервер перегружен.
2. Почему? → 5000 одновременных запросов.
3. Почему? → Ботнет-атака.
4. Почему не остановили? → WAF не сработал.
5. Почему? → WAF не обновляли 6 месяцев.
Корень: нет процесса регулярного обновления WAF.

## Часть B: Ishikawa (Fishbone)

#
## 📖 ИСТОРИЯ СОЗДАНИЯ

1950-е. Япония. Сакити Тоёда, основатель Toyota, формулирует принцип: «Спрашивай "почему?" пять раз». Его сын Кийтиро внедряет это в Toyota Production System.

Метод становится сердцем Toyota Production System и распространяется по всему миру как основа root cause analysis.

## 👤 КЛЮЧЕВЫЕ ФИГУРЫ

**Сакити Тоёда** (1867–1930) — основатель Toyota, изобрёл автоматический ткацкий станок.
**Каору Исикава** (1915–1989) — японский профессор, создатель диаграммы Исикавы и кружков качества.

## 🌍 КОНТЕКСТ ПРИМЕНЕНИЯ

- Производство: поиск корневой причины дефектов
- IT: анализ инцидентов и багов
- Управление: почему проект опоздал?

## 🏭 ИЗВЕСТНЫЕ КЕЙСЫ

**Toyota:** Пять «почему?» о покраске кузова привели не к замене робота, а к чистке форсунок раз в смену. Экономия: $500K/год на одном заводе.

---

## Алгоритм
1. Нарисуй горизонтальную стрелку вправо → проблему в «голове рыбы».

2. Проведи 6 «костей» (категорий) под углом:
   - **Man** (люди): навыки, обучение, коммуникация, мотивация
   - **Method** (методы): процессы, процедуры, инструкции
   - **Machine** (оборудование): техника, софт, инфраструктура
   - **Material** (материалы): сырьё, данные, контент
   - **Measurement** (измерения): метрики, KPI, точность данных
   - **Environment** (среда): офис, рынок, законодательство, погода

3. Для каждой категории — мозговой штурм: все возможные причины.

4. Сгруппируй причины: какие связаны?

5. Выдели 2-3 наиболее вероятные корневые причины.

6. Проверь их через Five Whys.

## Формат вывода
- **Проблема:** [формулировка]
- **Five Whys:** цепочка 1→2→3→4→5 [корень]
- **Fishbone:** для каждой из 6M — 2-3 причины
- **Корневая причина:** [что исправлять]
- **Действие:** [конкретный шаг]
