# Soft Systems Methodology (SSM) — Мягкая методология систем

**Автор:** Питер Чекленд (Lancaster University), 1970-е
**Тип:** Решение сложных организационных проблем с разными точками зрения
**Сложность:** ★★★★★ (экспертная)


## 📖 ИСТОРИЯ СОЗДАНИЯ

1970-е. Профессор Ланкастерского университета Питер Чекленд пытается применить инженерные методы к организационным проблемам. И терпит неудачу. Люди — не шестерёнки.

Чекленд разрабатывает Soft Systems Methodology — подход для «мягких», человеческих проблем, где участники даже не могут договориться, в ЧЁМ проблема. SSM не ищет «правильное решение» — она ищет договорённости.

## 👤 ОБ АВТОРЕ

**Питер Чекленд** (р. 1930) — британский учёный, профессор системного мышления. Его SSM преподают в бизнес-школах по всему миру.

## 🌍 КОНТЕКСТ ПРИМЕНЕНИЯ

- Организационные конфликты
- IT-проекты с рассогласованием требований
- Межведомственное взаимодействие

## 🏭 ИЗВЕСТНЫЕ КЕЙСЫ

**UK National Health Service:** SSM разрешила конфликт между врачами и администраторами о приоритетах цифровизации.
**Shell:** Применила SSM для согласования стратегии устойчивого развития со стейкхолдерами.

---

## КОГДА ПРИМЕНЯТЬ
- «Злая» проблема (wicked problem): нет согласия даже о том, в ЧЁМ проблема
- Разные стейкхолдеры видят ситуацию по-разному
- IT-проект буксует не из-за технологий, а из-за «политики»
- Организационная трансформация

## СУТЬ МЕТОДА
SSM признаёт: в человеческих системах нет одной объективной реальности. Есть РАЗНЫЕ Weltanschauung (мировоззрения). Цель SSM — не найти «правильное» решение, а достичь общих договорённостей (accommodations).

7 шагов:
1. Проблемная ситуация (неструктурированная)
2. Проблемная ситуация выражена (Rich Picture)
3. Root definitions (CATWOE для каждой перспективы)
4. Концептуальные модели (что нужно делать для каждой root definition)
5. Сравнение моделей с реальностью (дебаты)
6. Желаемые и осуществимые изменения
7. Действия по улучшению

## АЛГОРИТМ

### Шаг 1. Rich Picture
Нарисуй (словами) ситуацию: кто участники, какие конфликты, какие структуры.

### Шаг 2. Root Definitions (CATWOE)
Сформулируй 2-3 разные root definitions — КАЖДУЮ с позиции разного стейкхолдера.

### Шаг 3. Концептуальные модели
Для каждой root definition: какие действия должна выполнять система?

### Шаг 4. Дебаты
Сравни модели с реальностью. Где расхождения? Обсуди со стейкхолдерами.

### Шаг 5. Договорённости
Какие изменения системно желаемы (systemically desirable) и культурно осуществимы (culturally feasible)?
