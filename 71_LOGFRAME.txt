# Logical Framework Approach (LogFrame) — Логический фреймворк

**Автор:** USAID (1969), стандартизирован Всемирным банком и EC
**Тип:** Проектное планирование с измеримыми индикаторами
**Сложность:** ★★★☆☆ (средняя)


## 📖 ИСТОРИЯ СОЗДАНИЯ

1969 год. USAID (Агентство США по международному развитию) нуждается в инструменте планирования для проектов в развивающихся странах. Создаётся Logical Framework Approach (LogFrame).

Матрица 4×4: Goal, Purpose, Outputs, Activities × Narrative, Indicators, Verification, Assumptions. Идея: вся логика проекта на ОДНОМ ЛИСТЕ.

LogFrame становится стандартом для Всемирного банка, ЕС, ООН.

## 👤 ОБ АВТОРЕ

Разработан USAID (1969). Стандартизирован в Project Cycle Management (PCM) Европейской комиссии.

## 🌍 КОНТЕКСТ ПРИМЕНЕНИЯ

- Грантовые заявки
- Международные проекты развития
- Бюджетирование программ

## 🏭 ИЗВЕСТНЫЕ КЕЙСЫ

**Всемирный банк:** LogFrame — обязательный формат для всех проектов ($60+ млрд портфель). Стандарт с 1970-х.

---

## КОГДА ПРИМЕНЯТЬ
- Планирование крупного проекта (грант, госпрограмма, aid)
- Нужна логическая матрица на одном листе
- Требование донора (USAID, EU, World Bank)
- Связь между целями, действиями и бюджетом

## СУТЬ МЕТОДА
LogFrame = матрица 4×4:

| | Narrative | Indicators | Verification | Assumptions |
|---|----------|-----------|-------------|-------------|
| **Goal (Цель)** | долгосрочное воздействие | KPI | источник данных | — |
| **Purpose (Назначение)** | что изменится | KPI | источник | если Purpose → Goal |
| **Outputs (Результаты)** | что произведём | KPI | источник | если Outputs → Purpose |
| **Activities (Действия)** | что делаем | ресурсы | бюджет | если Activities → Outputs |

Вертикальная логика: если Действия выполнены → Результаты достигнуты → Назначение выполнено → Цель достигнута.

## АЛГОРИТМ

### Шаг 1. Спускайся сверху вниз (цели)
Goal → Purpose → Outputs → Activities.

### Шаг 2. Проверь логику снизу вверх
Если Activities выполнены, ДОСТАТОЧНО ли этого для Outputs? И так вверх.

### Шаг 3. Заполни Assumptions
Что должно быть истинным ВНЕ проекта для успеха?

### Шаг 4. Определи Indicators и Verification
SMART-индикаторы (Specific, Measurable, Achievable, Relevant, Time-bound).
