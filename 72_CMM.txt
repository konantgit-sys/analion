# CMM/CMMI — Модель зрелости процессов

**Автор:** Software Engineering Institute (Carnegie Mellon), 1987-2018
**Тип:** Оценка и улучшение зрелости организационных процессов
**Сложность:** ★★★★☆ (продвинутая)


## 📖 ИСТОРИЯ СОЗДАНИЯ

1987 год. Министерство обороны США сталкивается с катастрофическим качеством софта в военных системах. Software Engineering Institute (Carnegie Mellon) получает задание: создать метод ОЦЕНКИ подрядчиков.

Уоттс Хамфри разрабатывает Capability Maturity Model — шкалу от 1 (хаос) до 5 (оптимизация). Идея: нельзя прыгнуть с уровня 1 на 5. Нужно последовательно пройти каждый уровень.

К 2000-м CMMI становится мировым стандартом для софтверных компаний, желающих работать с правительством США.

## 👤 ОБ АВТОРЕ

## СУТЬ МЕТОДА
5 уровней зрелости:

- **Level 1 — Initial (Начальный):** хаос, герои-одиночки, успех зависит от людей, не от процессов.
- **Level 2 — Managed (Управляемый):** проекты планируются, отслеживаются, контролируются. Но процессы разные.
- **Level 3 — Defined (Определённый):** единые стандартные процессы на всю организацию. Tailoring guidelines.
- **Level 4 — Quantitatively Managed (Количественно):** процессы измеряются метриками, статистическое управление.
- **Level 5 — Optimizing (Оптимизирующий):** непрерывное улучшение на основе данных, инновации.

## АЛГОРИТМ ОЦЕНКИ

По объективным признакам для каждого process area.

### Шаг 2. Определи целевой уровень
Level 3 — минимум для стабильного качества. Level 5 — для высококонкурентных рынков.

### Шаг 3. Gap Analysis
Какие process areas нужно улучшить для перехода на следующий уровень?

### Шаг 4. Дорожная карта
Приоритетные области, действия, метрики прогресса.
