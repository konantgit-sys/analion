# Gap Analysis — Анализ разрывов

**Автор:** Метод стратегического управления (1960-е)
**Тип:** Сравнение текущего и желаемого состояния
**Сложность:** ★★☆☆☆ (базовая)


## 📖 ИСТОРИЯ СОЗДАНИЯ

1960-е. Стратегическое планирование. Консультанты формализуют Gap Analysis — метод сравнения ТЕКУЩЕГО состояния с ЖЕЛАЕМЫМ. Разрыв (gap) = то, что нужно преодолеть.

Gap Analysis становится фундаментом стратегического планирования: где мы сейчас? → Где хотим быть? → Какой разрыв? → Как его закрыть?

## 👤 ОБ АВТОРЕ

Метод развивался эволюционно в консалтинговой практике 1960-70-х (McKinsey, BCG).

## 🌍 КОНТЕКСТ ПРИМЕНЕНИЯ

- Стратегия: разрыв между текущей и целевой выручкой
- IT: разрыв между текущим и целевым уровнем сервиса
- Навыки: разрыв в компетенциях команды

## 🏭 ИЗВЕСТНЫЕ КЕЙСЫ

**Amazon (2000-е):** Gap-анализ показал разрыв между текущим IT и потребностями при росте. → AWS — закрыл gap и создал новый бизнес на $90 млрд.

## КОГДА ПРИМЕНЯТЬ
- Непонятно, почему не достигаются цели
- Планирование: что нужно сделать для перехода из точки А в точку Б?
- Оценка готовности к изменениям
- Бенчмаркинг: сравнение с лидером рынка

## СУТЬ МЕТОДА
Gap = разрыв между текущим состоянием и желаемым.

3 типа разрывов:
- **Performance gap:** делаем хуже, чем планировали
- **Opportunity gap:** делаем хорошо, но рынок растёт быстрее
- **Strategic gap:** мы не там, где хотим быть стратегически

Формула: Gap = Desired State − Current State

## АЛГОРИТМ АНАЛИЗА

### Шаг 1. Опиши текущее состояние (As-Is)
Конкретные метрики. Не «плохо», а «выручка 10М, рост 5% в год, NPS 30».

### Шаг 2. Опиши желаемое состояние (To-Be)
Конкретные метрики с дедлайном. «Выручка 25М, рост 15%, NPS 60 через 2 года».

### Шаг 3. Измерь разрыв (Gap)
Разница по каждой метрике.

### Шаг 4. Корневые причины разрыва
Почему существует разрыв? Чего не хватает: компетенций, ресурсов, рынка, стратегии?

### Шаг 5. План закрытия разрыва
Мост из As-Is в To-Be:
- Что нужно начать делать?
- Что нужно перестать делать?
- Что нужно делать иначе?
