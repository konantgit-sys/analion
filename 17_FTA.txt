# Fault Tree Analysis — Анализ дерева отказов

**Автор:** Bell Labs (1962), для ВВС США (программа Minuteman)
**Тип:** Дедуктивный логический анализ отказов
**Что анализирует:** Причинно-следственные цепочки, ведущие к отказу системы

## Контекст
Если FMEA смотрит «снизу вверх» (компонент → последствия), то FTA — 
«сверху вниз» (отказ системы → первопричины). 
Метод родился в Bell Labs для анализа безопасности межконтинентальных 
баллистических ракет Minuteman. Сегодня обязателен в ядерной энергетике, 
авиации (ARP4761) и космонавтике.


## 📖 ИСТОРИЯ СОЗДАНИЯ

1962 год. Лаборатории Bell Telephone. Инженер Х.А. Уотсон разрабатывает метод для анализа надёжности системы управления запуском ракеты Minuteman. Задача: что может привести к ОТКАЗУ запуска?

Fault Tree Analysis (FTA) использует булеву логику: событие «отказ системы» раскладывается на комбинации отказов компонентов через логические элементы AND/OR. Дерево растёт сверху вниз — от нежелательного события к первопричинам.

После катастрофы Challenger (1986) FTA становится обязательным для NASA. Сегодня это стандарт в аэрокосмической, ядерной и химической промышленности.

## 👤 ОБ АВТОРЕ

**Х.А. Уотсон** — инженер Bell Labs, разработал FTA по заказу ВВС США для программы межконтинентальных баллистических ракет Minuteman.

## 🌍 КОНТЕКСТ ПРИМЕНЕНИЯ

- Аэрокосмическая промышленность
- Ядерная безопасность
- Химическое производство

## 🏭 ИЗВЕСТНЫЕ КЕЙСЫ

**Challenger (1986):** FTA системы твердотопливных ускорителей выявил бы, что O-кольца при низких температурах — single-point failure всей системы.
**Fukushima (2011):** FTA показал, что волна цунами > проектной высоты стены — отказ, который не был учтён.

## Алгоритм анализа

### Шаг 1: Определи ВЕРХНЕЕ НЕЖЕЛАТЕЛЬНОЕ СОБЫТИЕ
Сформулируй чётко: «Система X отказала способом Y».
Примеры: «Двигатель не запустился», «Крыло потеряло несущую способность», «Данные клиентов утекли».

### Шаг 2: Строй дерево событий сверху вниз
Основной вопрос на каждом шагу: «КАКИМ ОБРАЗОМ это может произойти?»
Используй логические вентили:
- **AND-вентиль:** событие происходит только если ВСЕ входящие события истинны (∩)
- **OR-вентиль:** событие происходит если ХОТЯ БЫ ОДНО входящее истинно (∪)

Рисуй схему: Верхнее событие → промежуточные события → базовые события (первопричины).

### Шаг 3: Дойди до базовых событий
Останавливай декомпозицию когда:
- Событие больше не раскладывается на под-события (атомарная причина)
- Дальнейший анализ даёт diminishing returns (убывающую пользу)
- Причина находится вне границ системы

### Шаг 4: Вычисли вероятность верхнего события
- Для AND: P(верх) = P(A) × P(B) × ... (все должны случиться)
- Для OR: P(верх) = 1 − (1−P(A)) × (1−P(B)) × ... (хотя бы одно)
- При малых вероятностях OR ≈ P(A) + P(B) + ...

### Шаг 5: Найди минимальные сечения (Minimal Cut Sets)
Минимальное сечение — наименьший набор базовых событий, при котором 
верхнее событие гарантированно происходит.
Упорядочи сечения по количеству элементов: одно-элементные сечения — 
самые критичные (одна точка отказа = отказ всей системы).

### Шаг 6: Предложи контрмеры
Для каждого минимального сечения:
- Можно ли устранить одно из событий полностью?
- Можно ли добавить AND-условие (дублирование, резервирование)?
- Можно ли снизить вероятность базовых событий?

## Формат вывода
Дерево отказов: AND/OR схема + таблица базовых событий с вероятностями + ранжированные минимальные сечения.
